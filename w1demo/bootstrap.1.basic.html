    Form - 表单
    表单是用户提交请求(与服务器交互)的重要手段.
    表单的工作方式:
    • 用户使用表单, 对服务器资源(图片, 网页, 博客等, named resource)进行访问
    • 表单本身有两个重要attributes
      • method: HTTP method, 用来告诉浏览器如何提交(submit)该表单, 主要有以下几种方式
        - get: 查询resource (query), 使用该方法时, 查询参数被加入到url中
        - post: 创建resource, 使用该方法时, 查询参数被放入query body中, 可以进行加密
        - put: Update/Replace resource.
        - patch: 只更新某个resource的一部分信息
        - delete: 删除resource, 这个一般在Restful中才会使用, 后边用到时详细介绍
      • action: 一个url, 指定了该表单会交给服务器的哪个函数/方法/程序进行处理.

        这个地方就是前后端需要沟通并制定API标准的地方.

    • 表单内包含输入项, 每个输入项必须有一个name属性, 用来标记服务器如何识别该输入项.
      { "userid":"Abc", ... }
  • Demo: basic
    • 常用class:
    • form-control/form-control-lg/form-control-sm: 几乎所有input element都应该加上这个class, 使得它符合bootstrap的样式风格(更好看)
    • readonly/form-control-plaintext:readonly属性使得Input框变成只读的;form-control-plaintext类使得输入框只读, 而且输入框看上去就像是一个普通的文字段落
    • form-group: 与card-group相似, 目的是更好的组织多个Input

  • Demo: 导航栏
    • 导航栏是一种特殊的导航条, 一般放在网页的最上方作为不同功能的入口.
    导航栏一般提供的功能更多, 例如brand, 搜索, 用户详情, admin入口等
    导航栏是针对整个网页的, 因此一般也会固定在顶部/底部:
    • .fixed-top: 整体页面的最上方
    • .fixed-bottom: 整体页面的最下方
    • .sticky-top : 父元素的最上方
    定位

    响应式导航栏, 当screen很小时将导航栏隐藏为一个可点击的下拉框
    配置重点:
    • 展开的按钮需要有一个.navbar-toggler类和data-target属性(该属性的值 = nav list div的id)
    • 需要隐藏的部分需要.collapse 和 .navbar-collapseclass
    • 在nav元素上使用.navbar-expand{-sm|-md|-lg|-xl}class来指定什么时候不隐藏
      (所以如果想让导航栏一直隐藏, 就不加任何.navbar-expand class)
